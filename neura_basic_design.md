# 基本設計書

> **プロジェクト名**：Neura
> **対象フェーズ**：Phase 1（MVP）モック実装用
> **このドキュメントの用途**：Claude Codeがモックを実装するための仕様書。API接続・認証・DB・本実装は対象外。
> **次のステップ**：このドキュメントをClaude Codeに渡してモックを実装する → モック確認後に detailed_design.md を作成する
> **関連ドキュメント**：
> - `neura_requirements.md`（機能要件・非機能要件 ← FR番号・エラー定義の出典）
> - `neura_architecture.md`（技術スタック・システム構成 ← 実装技術・ディレクトリの出典）

## 0. Claude Codeへのモック実装指示

### 実装技術

- フレームワーク：**バニラHTML + CSS + JavaScript（フレームワークなし）**
- スタイリング：**plain CSS（外部ライブラリなし）**
- ダミーデータ：**`docs/index.html` 内の `<script>` タグにハードコードする**
- ビルドステップ：**なし**（ファイルをブラウザで直接開いて確認する）

### 実装対象

以下の画面を `docs/index.html` の **1ファイル**にまとめて実装すること。

| # | 画面名 | 対応FR | 優先度 |
|---|---|---|---|
| 1 | SCR-01：ホーム（日付一覧） | FR-05 | 高 |
| 2 | SCR-02：日次詳細（記事一覧）＋全文翻訳パネル | FR-05 | 高 |
| 3 | SCR-03：キーワード検索結果 | FR-05 | 高 |
| 4 | SCR-04：設定画面 | FR-06 | 中 |

### モックで実装しないこと（重要）

- **API接続・fetch呼び出し**：すべてハードコードのダミーデータを使用すること。実際の `docs/data/*.json` をfetchしないこと
- **GitHub Actions・Pythonスクリプト**：FR-01〜FR-04はバックエンドバッチ処理のためモック対象外
- **Discord通知**：FR-03はモック対象外
- **GitHub Contents API（FR-06）**：SCR-04の**レイアウト・フォーム・各状態（PAT未設定／読込中／保存成功・失敗トースト）はモックで実装する**が、実際のGitHub APIへの読み書きは本実装フェーズで行う。モックでは `DEFAULT_CONFIG_JS` 相当のダミー設定をフォームに表示し、「保存する」押下時はトーストのみ表示する（実通信なし）
- **リアルタイム更新**：1日1回バッチのためリアルタイム更新機能は実装しないこと
- **認証**：不要（個人利用。ただしSCR-04のPAT入力欄UIはモックで実装する）

> ※ SCR-03（キーワード検索）はモックの実装対象。ダミーデータに対してクライアント側JSでAND検索を行う。
> ※ SCR-04（設定画面）はUI・状態遷移をモック対象とし、GitHub APIの実通信のみ本実装に委ねる（SCR-01〜03のfetchと同じ扱い）。

### ファイル構成の指示

```
neura/
└── docs/
    └── index.html    ← このファイル1つだけを作成すること
```

- SCR-01〜SCR-04の全画面を **`docs/index.html` の1ファイルに実装**すること
- URLパラメータ（`?date=YYYY-MM-DD` / `?q=キーワード` / `?view=settings`）で画面を切り替えること
- CSSは `<style>` タグ内に記述すること
- JavaScriptは `<script>` タグ内に記述すること
- 外部CDN・npmパッケージは使用しないこと
- **markdownレンダリング**：外部ライブラリは使用せず、`<script>` 内にインライン実装すること（コードブロック・インラインコード・太字・段落に対応）

### スタイリング方針

**デザインコンセプト**：ターミナル / ハッカー美学。AIニュースを扱う開発者向けサービスとして「CLIのような硬質で技術的な世界観」を採用する。黒背景にネオングリーンのアクセント、等幅フォントで統一する。

- **カラーパレット（CSS変数として `:root` に定義する）**：

  | 変数 | 値 | 用途 |
  |---|---|---|
  | `--bg` | `#080808` | ページ背景（ほぼ黒） |
  | `--surface` | `#0F0F0F` | ヘッダー・詳細/ホームヘッダーブロック背景 |
  | `--surface2` | `#161616` | カード内のネスト要素・コードブロック等 |
  | `--border` | `#222222` | 通常のボーダー |
  | `--border-mid` | `#333333` | やや強調するボーダー（タイル枠・ボタン枠） |
  | `--accent` | `#00FF41` | アクセント（ネオングリーン）。プロンプト記号・強調・LATEST・左ボーダー |
  | `--accent-dim` | `rgba(0,255,65,0.07)` | アクセントの極薄背景（ホバー等） |
  | `--accent-mid` | `rgba(0,255,65,0.14)` | アクセントの中間背景 |
  | `--text` | `#CCCCCC` | 本文テキスト |
  | `--text-bright` | `#EEEEEE` | 見出し・強調テキスト |
  | `--sub` | `#555555` | サブテキスト（ラベル・補助） |
  | `--sub-light` | `#3A3A3A` | さらに薄い装飾線・空状態 |
  | `--error` | `#FF4444` | エラー |
  | `--warn` | `#FFAA00` | 警告 |
  | `--news-c` | `#60A5FA`（青） | カテゴリ：ニュース |
  | `--research-c` | `#C084FC`（紫） | カテゴリ：研究 |
  | `--usecase-c` | `#34D399`（緑） | カテゴリ：活用事例 |
  | `--tool-c` | `#FBBF24`（黄） | カテゴリ：ツール |

- **フォント**：等幅フォントスタック `'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'SF Mono', 'Menlo', 'Consolas', monospace`（外部CDN不使用。ローカル/システム等幅フォントにフォールバック）。基本 `font-size: 13px` / `line-height: 1.6`。
- **デザイン方針**：
  - ダークターミナル基調。プロンプト記号（`>` / `$` / `#`）や `[ ]` 角括弧などCLIモチーフを装飾に使う
  - 見出し・重要要素はネオングリーン（`--accent`）＋ごく弱い発光（`text-shadow`）で強調する
  - ヘッダーブロックは `--surface` 背景＋左に3pxのグリーンボーダーで記事リストと差別化する
  - 情報密度は高めだが、余白とボーダーで領域を明確に分ける
- **レスポンシブ**：モバイル（375px）でも文字が切れず崩れないようにすること
- **アニメーション**：ホームヘッダーにニューラルネットワーク、詳細ヘッダーにオシロスコープ波形のCanvasアニメーションを配置する（`prefers-reduced-motion` で停止）。SCR-04にも同系統の控えめな演出を任意で付与してよい。

## 4. 画面仕様

---

### SCR-01：ホーム（日付一覧）

**対応FR**：FR-05（Webサイト閲覧）
**概要**：過去の日次ダイジェストを日付カードで一覧表示する。最新日付が上に来るよう降順で並べる。

#### レイアウト構造

```
┌──────────────────────────────────────────┐
│  🧠 Neura                                │  ← ヘッダー
│  AI Daily Digest                         │
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │  2026/06/18（水）  ●  7件          │  │  ← 日付カード（クリック可）
│  └────────────────────────────────────┘  │
│  ┌────────────────────────────────────┐  │
│  │  2026/06/17（火）  ●  8件          │  │
│  └────────────────────────────────────┘  │
│  ┌────────────────────────────────────┐  │
│  │  2026/06/16（月）  ●  5件          │  │
│  └────────────────────────────────────┘  │
│  ...（最新30件まで表示）                  │
└──────────────────────────────────────────┘
```

#### コンポーネント一覧

| コンポーネント名 | 種別 | 説明 |
|---|---|---|
| ヘッダー | ヘッダー | 「🧠 Neura」ロゴと「AI Daily Digest」サブタイトル |
| 日付カード | カード（クリック可） | 日付・曜日・記事件数を表示。クリックでSCR-02へ遷移 |

#### 表示データ（ダミーデータの定義）

| フィールド名 | 型 | ダミー値の例 | 説明 |
|---|---|---|---|
| `date` | string | `"2026-06-18"` | ISO 8601形式の日付（YYYY-MM-DD） |
| `displayDate` | string | `"2026/06/18（水）"` | カードに表示する日本語形式の日付 |
| `articleCount` | number | `7` | その日の記事件数 |

#### ユーザー操作と画面の変化

| 操作 | 変化・遷移先 |
|---|---|
| 日付カードをクリック | URLに `?date=YYYY-MM-DD` を付加してSCR-02へ切り替える |

#### 空の状態
- データが0件の場合：「まだデータがありません。しばらくお待ちください。」を中央に表示する

#### エラー状態
- `index.json` の取得失敗（モックでは発生しないが定義する）：「データの読み込みに失敗しました。しばらく後に再試行してください。」を中央に表示する

### SCR-02：日次詳細（記事一覧）

**対応FR**：FR-05（Webサイト閲覧）
**概要**：選択した日のAI記事を重要度降順でカード表示する。各カードから元記事へリンクできる。

#### レイアウト構造

```
┌──────────────────────────────────────────┐
│  🧠 Neura                                │  ← ヘッダー
├──────────────────────────────────────────┤
│  ← 一覧に戻る                            │  ← 戻るボタン
│  2026/06/18（水）のAIニュース  7件        │  ← ページタイトル
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │  🗞️ ニュース                       │  │  ← 記事カード
│  │  OpenAI、GPT-5を正式リリース        │  │
│  │  OpenAIが次世代モデルGPT-5を発表。  │  │
│  │  推論能力が大幅に向上し...          │  │
│  │  [HackerNews]  → 元記事を読む ↗   │  │
│  └────────────────────────────────────┘  │
│  ┌────────────────────────────────────┐  │
│  │  💡 活用事例                       │  │
│  │  ZennでClaude APIを使った業務自動化 │  │
│  │  請求書処理をAIで自動化した実装例。 │  │
│  │  [Zenn 🇯🇵]  → 元記事を読む ↗    │  │
│  └────────────────────────────────────┘  │
│  ...                                      │
└──────────────────────────────────────────┘
```

#### コンポーネント一覧

| コンポーネント名 | 種別 | 説明 |
|---|---|---|
| ヘッダー | ヘッダー | SCR-01と共通 |
| 戻るボタン | ボタン | クリックでSCR-01へ戻る（`?date=` パラメータを除去） |
| ページタイトル | テキスト | `{日付}のAIニュース　{件数}件` |
| 記事カード | カード | カテゴリバッジ・日本語タイトル・日本語要約・ソース名・全文翻訳ボタン・元記事リンクを表示 |
| カテゴリバッジ | バッジ | カテゴリに応じた絵文字＋テキスト |
| 全文翻訳ボタン | トグルボタン | クリックで翻訳パネルを展開/折りたたむ。`translation_ja` が `null` の場合は「翻訳取得できませんでした」を表示 |
| 翻訳パネル | 展開パネル | 記事の全文日本語翻訳をmarkdown形式でレンダリングして表示。コードブロックはシンタックスハイライト付きで表示 |
| 元記事リンク | リンク | 外部リンク（`target="_blank" rel="noopener noreferrer"`）で元記事を開く |

#### カテゴリバッジの定義

| カテゴリ値 | 表示 |
|---|---|
| `"ニュース"` | 🗞️ ニュース |
| `"研究"` | 🔬 研究 |
| `"活用事例"` | 💡 活用事例 |
| `"ツール"` | 🛠️ ツール |

#### ソース名の表示ルール

| source値 | 表示文字列 |
|---|---|
| `"HackerNews"` | `HackerNews` |
| `"Reddit"` | `Reddit` |
| `"RSS"` | `RSS` |
| `"Zenn"` | `Zenn 🇯🇵` |
| `"HatenaBookmark"` | `はてブ 🇯🇵` |

#### 表示データ（ダミーデータの定義）

| フィールド名 | 型 | ダミー値の例 | 説明 |
|---|---|---|---|
| `date` | string | `"2026-06-18"` | ISO 8601形式の日付 |
| `generated_at` | string | `"2026-06-18T04:00:00Z"` | 生成日時（UTC） |
| `articles` | Article[] | 下記参照 | 記事オブジェクトの配列 |
| `articles[].title_ja` | string | `"OpenAI、GPT-5を正式リリース"` | 日本語タイトル（30文字以内） |
| `articles[].summary_ja` | string | `"OpenAIが次世代モデルGPT-5を発表。推論能力が大幅に向上し..."` | 日本語要約（80文字以内） |
| `articles[].category` | string | `"ニュース"` | `"ニュース"` / `"研究"` / `"活用事例"` / `"ツール"` |
| `articles[].importance` | number | `5` | 重要度スコア（1〜5、5が最重要） |
| `articles[].url` | string | `"https://example.com/article"` | 元記事URL |
| `articles[].source` | string | `"HackerNews"` | 情報源名 |
| `articles[].published_at` | string | `"2026-06-18T03:00:00Z"` | 元記事の公開日時（UTC） |

#### ユーザー操作と画面の変化

| 操作 | 変化・遷移先 |
|---|---|
| 「← 一覧に戻る」をクリック | SCR-01へ切り替える（URLから `?date=` パラメータを除去） |
| 「▼ 全文翻訳を見る」をクリック | 翻訳パネルが展開され、ボタンが「▲ 閉じる」に変わる |
| 「▲ 閉じる」をクリック | 翻訳パネルが折りたたまれ、ボタンが「▼ 全文翻訳を見る」に戻る |
| 「元記事を読む」をクリック | 新しいタブで元記事URLを開く |

#### 空の状態
- 記事が0件の場合：「この日のデータが見つかりません。」を中央に表示する

#### エラー状態
- 存在しない日付を指定した場合（`?date=2000-01-01` 等）：「指定した日付のデータが見つかりません。」を中央に表示し、「← 一覧に戻る」ボタンを表示する
- 当日13時前（JSONがまだ生成されていない）：同上のエラーメッセージを表示する

| Owner / Repo 入力欄 | 入力 | 値を `localStorage` に保存（PATとは別に `neura_github_owner` / `neura_github_repo` キーで保存） |

| ジャンルチェックボックス | ON/OFF | 未保存の変更として記憶（「保存する」押下時にまとめて送信） |
| ソースチェックボックス | ON/OFF | 同上 |
| 「+ ソースを追加」ボタン | クリック | name / url の入力行を1行追加する |
| 実行時刻セレクトボックス | 選択（0〜23時） | 未保存の変更として記憶 |
| キーワード入力欄 | 入力 | カンマ区切りで編集。未保存の変更として記憶 |
| Geminiプロンプト textarea | 入力 | 全文をテキストエリアで直接編集。未保存の変更として記憶 |
| 「保存する」ボタン | クリック | GitHub Contents API に PUT して `config/config.json` を更新。`run_hour_jst` が変更された場合は `.github/workflows/daily.yml` の cron 式も更新。成功時トーストを表示 |

## 5. データモデル

### DigestIndex（日付一覧）

ホーム画面が1回のfetchで件数・カテゴリ分布バーを描画できるよう、各日付のメタ情報を保持する。

| フィールド名 | 型 | 必須 | 説明 | ダミー値の例 |
|---|---|---|---|---|
| `digests` | DigestMeta[] | ✅ | 日付メタ情報の降順配列、最新100件まで | 下記参照 |
| `digests[].date` | string | ✅ | YYYY-MM-DD形式 | `"2026-06-18"` |
| `digests[].count` | number | ✅ | その日の記事件数 | `7` |
| `digests[].categories` | object | ✅ | カテゴリ別件数（存在するカテゴリのみ） | `{"ニュース":3,"研究":1,"活用事例":1,"ツール":2}` |

### DailyDigest（日次データ）

| フィールド名 | 型 | 必須 | 説明 | ダミー値の例 |
|---|---|---|---|---|
| `date` | string | ✅ | YYYY-MM-DD形式 | `"2026-06-18"` |
| `generated_at` | string | ✅ | ISO 8601 UTC | `"2026-06-18T04:00:00Z"` |
| `articles` | Article[] | ✅ | 記事オブジェクトの配列（5〜10件） | 下記参照 |

### Article（記事）

| フィールド名 | 型 | 必須 | 説明 | ダミー値の例 |
|---|---|---|---|---|
| `title_ja` | string | ✅ | 日本語タイトル（30文字以内） | `"OpenAI、GPT-5を正式リリース"` |
| `summary_ja` | string | ✅ | 日本語要約（80文字以内） | `"OpenAIが次世代モデルGPT-5を発表。推論能力が大幅に向上し..."` |
| `translation_ja` | string \| null | ✅ | 記事本文の全文日本語翻訳（markdown形式）。本文取得失敗時は `null` | 下記参照 |
| `category` | string | ✅ | `"ニュース"` / `"研究"` / `"活用事例"` / `"ツール"` | `"ニュース"` |
| `importance` | number | ✅ | 1〜5の整数（5が最重要） | `5` |
| `url` | string | ✅ | 元記事URL（`http://` or `https://` で始まる） | `"https://openai.com/blog/gpt-5"` |
| `source` | string | ✅ | `"HackerNews"` / `"Reddit"` / `"RSS"` / `"Zenn"` / `"HatenaBookmark"` | `"HackerNews"` |

**`translation_ja` フィールドの形式：**
- Geminiが出力するmarkdown文字列
- コードブロックは ` ```言語名 ``` ` 形式で保持（例：` ```python `）
- `null` の場合：本文取得がブロックされたか取得に失敗した記事。フロントエンドは「⚠️ この記事の翻訳を取得できませんでした」と表示する

### AppConfig（設定ファイル）

| `sources[].url` | string | ✅ | 収集元URL | — |
