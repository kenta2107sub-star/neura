# 品質・セキュリティルール（.claude/rules.md）

---

## 品質ルール

```
設計書との整合性：
  □ 設計書にないものを実装しない
  □ エラーメッセージ・エラーコードは requirements.md の定義と一致させる
  □ ファイルは architecture.md のディレクトリ構成通りに配置する

実装の進め方：
  □ 一度に1つの変更を行う
  □ テストがFAILしている状態でリファクタリング・デプロイをしない
  □ 同じアプローチで3回失敗したら別の方法を試す
  □ 5つの方法を試して解決しない場合はユーザーにエスカレーションする

コードの品質：
  □ 1ファイル1責務を維持する
  □ Route Handler にビジネスロジックを書かない
  □ デバッグ用ログを本番コードに残さない
```

---

## セキュリティルール

基本的な禁止事項（APIキーのハードコード禁止・.envのコミット禁止・機密情報のログ出力禁止）は
`~/CLAUDE.md`（グローバル）の「禁止事項」セクションで定義されている。

以下はそれに加えた実装レベルのセキュリティルール：

```
  □ 環境変数は必ず .env.local / .env から読み込む
  □ 認証が必要なエンドポイントには認証チェックを実装する
  □ 外部からの入力は必ずバリデーションする
  □ エラーメッセージにスタックトレースを含めない
```
