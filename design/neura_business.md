## 1. サービス概要

### サービス名
`Neura`（ニューラ）

### サービス名の由来
ニューラルネットワーク（neural）から着想。AIの神経回路のように、散在する情報を毎日1つに束ねて届けるというコンセプトを表す。

### 目的
AIの進化スピードが速すぎて毎日の情報収集が追いつかない個人開発者・AI活用者向けに、海外（HN・Reddit・RSS）と日本語（Zenn・はてなブックマーク）の両ソースを横断収集し、毎日13時に日本語で要約・Discordに届けるとともにWebアーカイブで過去記事を閲覧できるシステム。情報収集にかける時間をゼロにすることが目的。

### フェーズ計画

| フェーズ | 内容 | 状態 |
|---|---|---|
| Phase 1（MVP） | HN・Reddit・RSS・Zenn・はてブからAI記事を自動収集・Gemini要約・Discord通知・GitHub Pagesアーカイブ | **← 現在実装対象** |
| Phase 2 | YouTube動画要約の追加・カテゴリフィルタ・タグ検索 | 将来 / 検討中 |
| Phase 3 | X（Twitter）連携（公式API状況次第）・メール通知対応 | 将来 / 検討中 |

## 2. 競合分析

> 調査日：2026-06-18（Web検索で確認）

### 競合の分類

**① The Rundown AI・TLDR AI・Superhuman AI（英語圏の大手AIニュースレター）**
The Rundown AIは200万人超、TLDR AIは125万人超の読者を持つ英語ニュースレター。AIニュースのキュレーションという点では直接的な競合だが、**英語のみ・メール配信・個人カスタマイズ不可**という制約がある。

**② Readless・Particle・Feedly（AI活用ニュースアグリゲーター）**
AIでニュースを要約するWebサービス群。月額課金が多く、日本語ソース（Zenn・はてブ）への対応が薄い。

**③ 手動情報収集（Twitterを毎朝チェック・RSSリーダーを手動確認）**
最も多くの個人が現在取っている代替手段。情報量が多すぎて見落としが発生し、毎日時間がかかる。

### 差別化ポイント

- **日本語×日本語ソースの組み合わせ**：英語圏の大手ニュースレターはZenn・はてなブックマーク等の日本語ソースを一切拾えない。日本人エンジニアの活用事例は日本語ソースにしか存在しないため、他サービスでは代替できない
- **完全無料・広告なし**：大手ニュースレターは広告収入モデルのため記事選定に恣意性がある。Neuraは個人運用のため広告が一切入らない
- **Discordへの直接通知**：既存サービスはメール配信のみ。すでにDiscordを日常利用している開発者にとってはメールより気づきやすい

### 注意すべきリスク

- **Gemini Flash無料枠の廃止**：Googleが無料枠を廃止した場合、要約コストが発生する。対応策：無料枠廃止時はOpenAI互換の別の無料モデル（Groq等）に切り替えを検討
- **Reddit・はてなブックマークのAPI仕様変更**：無認証JSONエンドポイントが閉鎖された場合、収集スクリプトの修正が必要になる。対応策：収集ソースを複数持つことで単一ソースの障害に強い構成にしている

## 3. フェーズ計画と将来の拡張

### Phase 1（現在）：MVP

**スコープ（実装対象）**
- 情報収集：HN・Reddit×2・RSS×3・Zenn・はてなブックマーク
- AI要約：Gemini Flash（日本語・カテゴリ分類・重要度スコア）
- Discord通知：毎日13時、5〜10件
- GitHub Pagesアーカイブ：日次JSONの蓄積・閲覧

**スコープ外（Phase 1では作らない）**
- X・TikTok・YouTube連携
- カテゴリフィルタ・タグ検索
- メール通知

### Phase 2（将来）：情報源の拡充

| 追加内容 | 理由 |
|---|---|
| YouTube動画要約 | AI解説動画（Two Minute Papers等）の要約をAgent-Reach経由で追加 |
| カテゴリフィルタ | 「研究だけ見たい日」などユースケースに応じた絞り込み |
| タグ検索 | 過去アーカイブから特定キーワードで検索できる機能 |

### Phase 3（将来）：通知チャンネルの拡充

| 追加内容 | 条件 |
|---|---|
| X（Twitter）連携 | 公式APIが無料枠を再提供した場合に追加 |
| メール通知 | SendGrid無料枠（100通/日）を利用して追加 |

## 5. 運用停止フロー

Neuraの運用を停止する場合は以下の順序で対応する。

1. **GitHub Actionsを無効化する**：GitHubリポジトリ → Actions → 「Neura Daily Digest」ワークフロー → 「Disable workflow」をクリックする
2. **Discord Webhookを削除する**：Discord → `#ai-news` チャンネル → チャンネル編集 → 「連携サービス」→ 該当Webhookの「削除」をクリックする
3. **Gemini APIキーを削除する（任意）**：Google AI Studio → 「Get API key」→ 該当キーの削除アイコンをクリックする
4. **GitHubリポジトリをアーカイブする（任意）**：リポジトリ → Settings → 「Archive this repository」をクリックする（削除ではなくアーカイブ推奨。過去のアーカイブデータが残る）

> GitHub Pages・GitHub Actionsシークレットはリポジトリをアーカイブまたは削除すれば自動的に無効化される。
