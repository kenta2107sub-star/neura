# 基本設計書

> **プロジェクト名**：Neura
> **対象フェーズ**：Phase 1（MVP）モック実装用
> **このドキュメントの用途**：Claude Codeがモックを実装するための仕様書。API接続・認証・DB・本実装は対象外。
> **次のステップ**：このドキュメントをClaude Codeに渡してモックを実装する → モック確認後に detailed_design.md を作成する
> **関連ドキュメント**：
> - `neura_requirements.md`（機能要件・非機能要件 ← FR番号・エラー定義の出典）
> - `neura_architecture.md`（技術スタック・システム構成 ← 実装技術・ディレクトリの出典）

## 0. Claude Codeへのモック実装指示

### 実装技術

- フレームワーク：**バニラHTML + CSS + JavaScript（フレームワークなし）**
- スタイリング：**plain CSS（外部ライブラリなし）**
- ダミーデータ：**`docs/index.html` 内の `<script>` タグにハードコードする**
- ビルドステップ：**なし**（ファイルをブラウザで直接開いて確認する）

### 実装対象

以下の画面を `docs/index.html` の **1ファイル**にまとめて実装すること。

| # | 画面名 | 対応FR | 優先度 |
|---|---|---|---|
| 1 | SCR-01：ホーム（日付一覧） | FR-05 | 高 |
| 2 | SCR-02：日次詳細（記事一覧）＋全文翻訳パネル | FR-05 | 高 |
| 3 | SCR-03：キーワード検索結果 | FR-05 | 高 |
| 4 | SCR-04：設定画面 | FR-06 | 中 |

### モックで実装しないこと（重要）

- **API接続・fetch呼び出し**：すべてハードコードのダミーデータを使用すること。実際の `docs/data/*.json` をfetchしないこと
- **GitHub Actions・Pythonスクリプト**：FR-01〜FR-04はバックエンドバッチ処理のためモック対象外
- **Discord通知**：FR-03はモック対象外
- **GitHub Contents API（FR-06）**：SCR-04の**レイアウト・フォーム・各状態（PAT未設定／読込中／保存成功・失敗トースト）はモックで実装する**が、実際のGitHub APIへの読み書きは本実装フェーズで行う。モックでは `DEFAULT_CONFIG_JS` 相当のダミー設定をフォームに表示し、「保存する」押下時はトーストのみ表示する（実通信なし）
- **リアルタイム更新**：1日1回バッチのためリアルタイム更新機能は実装しないこと
- **認証**：不要（個人利用。ただしSCR-04のPAT入力欄UIはモックで実装する）

> ※ SCR-03（キーワード検索）はモックの実装対象。ダミーデータに対してクライアント側JSでAND検索を行う。
> ※ SCR-04（設定画面）はUI・状態遷移をモック対象とし、GitHub APIの実通信のみ本実装に委ねる（SCR-01〜03のfetchと同じ扱い）。

### ファイル構成の指示

```
neura/
└── docs/
    └── index.html    ← このファイル1つだけを作成すること
```

- SCR-01〜SCR-04の全画面を **`docs/index.html` の1ファイルに実装**すること
- URLパラメータ（`?date=YYYY-MM-DD` / `?q=キーワード` / `?view=settings`）で画面を切り替えること
- CSSは `<style>` タグ内に記述すること
- JavaScriptは `<script>` タグ内に記述すること
- 外部CDN・npmパッケージは使用しないこと
- **markdownレンダリング**：外部ライブラリは使用せず、`<script>` 内にインライン実装すること（コードブロック・インラインコード・太字・段落に対応）

### スタイリング方針

**デザインコンセプト**：ターミナル / ハッカー美学。AIニュースを扱う開発者向けサービスとして「CLIのような硬質で技術的な世界観」を採用する。黒背景にネオングリーンのアクセント、等幅フォントで統一する。

- **カラーパレット（CSS変数として `:root` に定義する）**：

  | 変数 | 値 | 用途 |
  |---|---|---|
  | `--bg` | `#080808` | ページ背景（ほぼ黒） |
  | `--surface` | `#0F0F0F` | ヘッダー・詳細/ホームヘッダーブロック背景 |
  | `--surface2` | `#161616` | カード内のネスト要素・コードブロック等 |
  | `--border` | `#222222` | 通常のボーダー |
  | `--border-mid` | `#333333` | やや強調するボーダー（タイル枠・ボタン枠） |
  | `--accent` | `#00FF41` | アクセント（ネオングリーン）。プロンプト記号・強調・LATEST・左ボーダー |
  | `--accent-dim` | `rgba(0,255,65,0.07)` | アクセントの極薄背景（ホバー等） |
  | `--accent-mid` | `rgba(0,255,65,0.14)` | アクセントの中間背景 |
  | `--text` | `#CCCCCC` | 本文テキスト |
  | `--text-bright` | `#EEEEEE` | 見出し・強調テキスト |
  | `--sub` | `#555555` | サブテキスト（ラベル・補助） |
  | `--sub-light` | `#3A3A3A` | さらに薄い装飾線・空状態 |
  | `--error` | `#FF4444` | エラー |
  | `--warn` | `#FFAA00` | 警告 |
  | `--news-c` | `#60A5FA`（青） | カテゴリ：ニュース |
  | `--research-c` | `#C084FC`（紫） | カテゴリ：研究 |
  | `--usecase-c` | `#34D399`（緑） | カテゴリ：活用事例 |
  | `--tool-c` | `#FBBF24`（黄） | カテゴリ：ツール |

- **フォント**：等幅フォントスタック `'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'SF Mono', 'Menlo', 'Consolas', monospace`（外部CDN不使用。ローカル/システム等幅フォントにフォールバック）。基本 `font-size: 13px` / `line-height: 1.6`。
- **デザイン方針**：
  - ダークターミナル基調。プロンプト記号（`>` / `$` / `#`）や `[ ]` 角括弧などCLIモチーフを装飾に使う
  - 見出し・重要要素はネオングリーン（`--accent`）＋ごく弱い発光（`text-shadow`）で強調する
  - ヘッダーブロックは `--surface` 背景＋左に3pxのグリーンボーダーで記事リストと差別化する
  - 情報密度は高めだが、余白とボーダーで領域を明確に分ける
- **レスポンシブ**：モバイル（375px）でも文字が切れず崩れないようにすること
- **アニメーション**：ホームヘッダーにニューラルネットワーク、詳細ヘッダーにオシロスコープ波形のCanvasアニメーションを配置する（`prefers-reduced-motion` で停止）。SCR-04にも同系統の控えめな演出を任意で付与してよい。

## 4. 画面仕様

---

### SCR-01：ホーム（日付一覧）

**対応FR**：FR-05（Webサイト閲覧）
**概要**：過去の日次ダイジェストを日付カードで一覧表示する。最新日付が上に来るよう降順で並べる。

#### レイアウト構造

月ごとにグループ化し、月ヘッダー（クリックで開閉）＋日付カードの2階層で表示する。最新月は自動展開、それ以外は折りたたみ状態で初期表示する。同一日付に複数スロット（時刻）の配信がある場合、日付ごとに1枚の日付タイルを共有しつつ、スロットごとに1枚のカードを縦に並べる。

```
┌──────────────────────────────────────────┐
│  🧠 Neura                                │  ← ヘッダー
│  AI Daily Digest                         │
├──────────────────────────────────────────┤
│  › 2026.06        18d / 132items          │  ← 月ヘッダー（クリックで開閉）
│  ┌────────────────────────────────────┐  │
│  │ [18] 06/18  19:00  LATEST          │  │  ← 日付カード（最新スロットのみ）
│  │      ▓▓▓▓░░  [07]  ›               │  │     カテゴリ分布バー＋件数＋タイトル一覧
│  │      1. OpenAI、GPT-5を正式リリース │  │
│  │      2. ...(先頭10件)               │  │
│  │      06/18  13:00                  │  │  ← 同日の別スロット
│  │      ▓▓▓░░░  [05]  ›               │  │
│  └────────────────────────────────────┘  │
│  › 2026.05        20d / 140items          │  ← 折りたたみ状態の月
└──────────────────────────────────────────┘
```

#### コンポーネント一覧

| コンポーネント名 | 種別 | 説明 |
|---|---|---|
| ヘッダー | ヘッダー | 「🧠 Neura」ロゴと「AI Daily Digest」サブタイトル |
| 月ヘッダー | 折りたたみトグル | `{年}.{月}` ＋ `{日数}d / {合計記事数}items`。クリックで配下のカード群を開閉。最新月のみ初期状態で展開 |
| 日付タイル | 装飾 | 同一日付の先頭カードにのみ表示する日付・曜日タイル。当日の最新スロットには `LATEST` タグを付与 |
| 日付カード | カード（クリック可） | 配信時刻・カテゴリ分布バー・記事件数・先頭10件のタイトル一覧（カテゴリバッジ付き）を表示。クリックで `goDetail(file)` を呼びSCR-02へ遷移。データが無い日は `[--]` 表示でクリック不可扱い |
| カテゴリ分布バー | バー | `categories` の内訳をカテゴリ別の色分けセグメントで表示。ホバーで件数入りツールチップを表示 |

#### 表示データ（ダミーデータの定義。`docs/data/index.json` の `digests` 要素に対応）

| フィールド名 | 型 | ダミー値の例 | 説明 |
|---|---|---|---|
| `date` | string | `"2026-06-18"` | ISO 8601形式の日付（YYYY-MM-DD） |
| `time` | string | `"19:00"` | 配信時刻（JST、`HH:00`） |
| `file` | string | `"2026-06-18_19"` | 対応する日次JSONのファイルキー。カードクリック時にこの値がそのまま `?date=` パラメータに使われる |
| `count` | number | `7` | そのスロットの記事件数 |
| `categories` | object | `{"ニュース": 3, "研究": 2}` | カテゴリ別件数（存在するカテゴリのみ） |
| `titles` | array | `[{"t": "OpenAI、GPT-5を正式リリース", "c": "ニュース"}, ...]` | 先頭10件のタイトル略称＋カテゴリ |

#### ユーザー操作と画面の変化

| 操作 | 変化・遷移先 |
|---|---|
| 月ヘッダーをクリック | 配下の日付カード群を開閉する（他の月の開閉状態には影響しない） |
| 日付カードをクリック | URLに `?date={file}`（例：`?date=2026-06-18_19`）を付加してSCR-02へ切り替える |

#### 空の状態
- データが0件の場合：「まだデータがありません。しばらくお待ちください。」を中央に表示する

#### エラー状態
- `index.json` の取得失敗（モックでは発生しないが定義する）：「データの読み込みに失敗しました。しばらく後に再試行してください。」を中央に表示する

### SCR-02：日次詳細（記事一覧）

**対応FR**：FR-05（Webサイト閲覧）
**概要**：選択した日のAI記事を重要度降順でカード表示する。各カードから元記事へリンクできる。

#### レイアウト構造

```
┌──────────────────────────────────────────┐
│  🧠 Neura                                │  ← ヘッダー
├──────────────────────────────────────────┤
│  ← cd ..                                 │  ← 戻るボタン（ターミナル風ラベル）
│  $ cat 2026/06/18（水） / AI digest       │  ← ページタイトル
├──────────────────────────────────────────┤
│  ┌────────────────────────────────────┐  │
│  │  🗞️ ニュース                       │  │  ← 記事カード（未読：通常表示）
│  │  OpenAI、GPT-5を正式リリース        │  │
│  │  OpenAIが次世代モデルGPT-5を発表。  │  │
│  │  推論能力が大幅に向上し...          │  │
│  │  [HackerNews]  [+ 全文翻訳]  → 元記事を読む ↗ │  │
│  └────────────────────────────────────┘  │
│  ┌────────────────────────────────────┐  │
│  │  ✓ 既読  💡 活用事例               │  │  ← 記事カード（既読：opacity 0.7）
│  │  ZennでClaude APIを使った業務自動化 │  │
│  │  請求書処理をAIで自動化した実装例。 │  │
│  │  [Zenn 🇯🇵]  [+ 全文翻訳]  → 元記事を読む ↗ │  │
│  └────────────────────────────────────┘  │
│  ...                                      │
└──────────────────────────────────────────┘

「[+ 全文翻訳]」クリック時に開くモーダル：
┌──────────────────────────────────────────┐
│  OpenAI、GPT-5を正式リリース    [ × 閉じる ] │
├──────────────────────────────────────────┤
│  ・推論能力が大幅に向上                    │
│  ・コンテキスト長が200万トークンに拡大      │
│  ・SWE-benchで85%を達成                   │
├──────────────────────────────────────────┤
│  ## OpenAI、GPT-5を正式リリース            │
│  OpenAIは本日...（markdownレンダリング）    │
└──────────────────────────────────────────┘
```

#### コンポーネント一覧

| コンポーネント名 | 種別 | 説明 |
|---|---|---|
| ヘッダー | ヘッダー | SCR-01と共通 |
| 戻るボタン | ボタン | ラベル「← cd ..」。クリックで `goHome()` を呼びSCR-01へ戻る（`?date=` パラメータを除去） |
| ページタイトル | テキスト | `{日付} / AI digest`（ターミナル風に `$ cat ` を疑似要素で前置する） |
| 記事カード | カード | カテゴリバッジ・日本語タイトル・日本語要約・ソース名・全文翻訳ボタン・元記事リンクを表示 |
| カテゴリバッジ | バッジ | カテゴリに応じた絵文字＋テキスト |
| 既読バッジ | バッジ | `localStorage` の `neura_read_articles` に記事URLが含まれる場合のみ、カード左上に `✓ 既読` を表示し、カード全体の `opacity` を `0.7` にする（FR-07）。SCR-03の記事カードにも同じ仕様を適用する |
| 全文翻訳ボタン | ボタン | ラベルは常に固定「[+ 全文翻訳]」（開閉で文言は変化しない）。クリックで全文翻訳モーダル（ダイアログ）を開き、その時点で既読フラグを立てる（FR-07）。`translation_ja` が `null` の場合は「翻訳取得できませんでした」を表示 |
| 全文翻訳モーダル | モーダルダイアログ | 画面中央にオーバーレイ表示。ヘッダーに記事タイトル＋「[ × 閉じる ]」ボタン。背景クリック・Escキー押下でも閉じる |
| 伝えたいことブロック | 箇条書き | モーダルのタイトル直下に表示。`key_points` を最大3件の箇条書きで表示する。`key_points` が空配列の場合はブロックごと非表示 |
| 翻訳パネル | 表示エリア | 伝えたいことブロックの下（モーダル内）に、記事の全文日本語翻訳をmarkdown形式でレンダリングして表示。コードブロックはシンタックスハイライト付きで表示 |
| 元記事リンク | リンク | 外部リンク（`target="_blank" rel="noopener noreferrer"`）で元記事を開く |

| `keywords.en` | string[] | ✅ | 英語フィルタリングキーワード | requirements.md FR-01 のデフォルトキーワード |
| `keywords.ja` | string[] | ✅ | 日本語フィルタリングキーワード | requirements.md FR-01 のデフォルトキーワード |
| `gemini_prompt` | string | ✅ | Gemini に送るプロンプト全文（設定UIには非公開。config.json 直接編集で変更可） | requirements.md FR-02 のデフォルトプロンプト |
| `notify_schedules` | NotifySchedule[] | ✅ | 通知スケジュール（最大3件） | スロット1のみ有効（13時・10件・全カテゴリ） |
| `notify_schedules[].hour` | number | ✅ | 実行時刻（JST 0〜23）。cron-job.org側のジョブがこの時刻でトリガーする（daily.ymlのcron式は書き換えない） | `13` / `20` / `8` |
| `notify_schedules[].enabled` | boolean | ✅ | スロットの有効/無効 | スロット1のみ `true` |
| `notify_schedules[].max_articles` | number | ✅ | そのスロットの通知件数上限（1〜10） | `10` |
| `notify_schedules[].genres` | object | ✅ | そのスロットで通知するカテゴリON/OFFマップ | 全カテゴリ `true` |
| `notify_schedules[].cron_job_id` | string \| null | ❌ | cron-job.org連携用のジョブID。Pythonスクリプトは未使用、設定画面のみ使用 | `null` |

> GitHub接続情報（owner / repo / PAT）・cron-job.org APIキーは config.json には含めず、ブラウザの `localStorage` に保存する（`neura_github_owner` / `neura_github_repo` / `neura_github_pat` / `neura_cronjob_apikey`）。理由は requirements.md FR-06 参照。
> グローバルの `genres` / `max_articles` / `run_hour_jst` は廃止済み。旧形式の config.json は `config_loader.py` が自動マイグレーションする。
