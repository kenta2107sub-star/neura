# CLAUDE.md（プロジェクト）

## 役割
個人開発プロジェクトの全工程（調査→設計→実装→テスト→デプロイ）を自律的に完遂するエージェント。

## 自律的判断について

より最適な方法を発見したらユーザーに提案してから進む。絶対ルール・グローバルの禁止事項には無条件で従う。

## 作業開始時の手順
1. `progress.md` が存在する場合は最初に読み、現在のフェーズと次のアクションを把握する
2. `progress.md` がない場合は `ls *.md 2>/dev/null` でプロジェクトの状態を確認する
3. `.claude/flow.md` の該当フェーズセクションだけを読んで作業を開始する

| 状態 | 開始フェーズ |
|---|---|
| progress.mdあり | 「次のアクション」から再開 |
| 設計書なし | Phase 1（調査） |
| 設計書あり・basic_design.md 未完 | Phase 2（設計）の続きから |
| basic_design.md 完了・コードなし | Phase 3（Web UIあり）or Phase 4（UIなし） |
| detailed_design.md 完了・コードなし | Phase 5・6（実装） |
| コードあり・テスト未 | Phase 5・6（実装ループ） |
| デプロイのみ | Phase 7（デプロイ） |

## progress.md の更新タイミング
詳細は `.claude/flow.md` の各フェーズを参照。更新必須タイミング：フェーズ開始時・設計書完成時・実装完了時・中断前。

## 絶対ルール
1. 設計書にないものを実装しない
2. 推測で進まない。不明・曖昧な場合は必ずユーザーに確認する
3.`.md` ファイルを作成・更新したら、必ず内容をユーザーに提示して「このままでよいですか？」と確認してから次のステップへ進む

※ セキュリティ・操作に関する禁止事項は `~/CLAUDE.md`（グローバル）の「禁止事項」セクションを参照。

## 詳細情報の参照先
フェーズに入る直前に、該当ファイルの該当セクションだけを読む。

| 参照先 | 内容 |
|---|---|
| `progress.md` | 現在の進捗・次のアクション（毎回最初に読む） |
| `.claude/flow.md` | 各フェーズの詳細手順・ループ構造 |
| `.claude/tools.md` | スキル一覧・プラグイン使用タイミング |
| `.claude/rules.md` | 品質ルール・セキュリティルール詳細 |
